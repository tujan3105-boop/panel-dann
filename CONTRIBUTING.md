# Contributing to GantengDann

Thanks for taking the time to contribute. This project is community driven, and every improvement helps.

## Code of Conduct
By participating, you agree to follow `CODE_OF_CONDUCT.md`.

## Before You Start
- Search existing issues to avoid duplicates.
- For security vulnerabilities, follow `SECURITY.md` and do not disclose publicly.

## Development Setup
The quickest path is:
- See the local setup steps in `README.md`.
- See frontend build details in `BUILDING.md`.

## Submitting Changes
1. Create a focused branch for your change.
2. Keep changes scoped and document any behavior changes.
3. Add or update tests when applicable.

## Pull Request Checklist
- The change has a clear purpose and scope.
- Tests or verification steps are included when relevant.
- Documentation updated when behavior or usage changes.

If you are unsure about an approach, open an issue first to discuss direction.
